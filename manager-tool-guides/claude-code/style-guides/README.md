# Style guides for Claude Code CLAUDE.md files

## Purpose

These are starter style guide files for use with Claude Code's `CLAUDE.md` configuration. They provide language-specific coding standards aligned with the [GDS Way programming language guidance](https://gds-way.digital.cabinet-office.gov.uk/standards/programming-languages.html#programming-languages).

Each file is a minimal viable starting point. Teams should adapt the contents to their project's specific conventions, frameworks, and tooling.

## How to use

Copy the relevant section into your project's `CLAUDE.md` file, or reference a style guide file directly by placing it in your repository.

For guidance on setting up `CLAUDE.md`, see the [customisation guide](../../../user-tool-guides/claude-code/customisation-guide.md).

## Principles

These style guides reflect the GDS Way principle that teams should:

- use the same language across a service where possible
- default to the most widely supported style guide for that language
- automate style checking using linters and formatters
- document exceptions to the style guide in the project's `CLAUDE.md`

# Training Materials

> Learning resources for AI Engineering Lab adoption across UK Government.

## Purpose

This section contains training materials designed to help government engineers, technical leads, and managers effectively adopt AI code assistants. Materials are designed to meet GOV.UK content standards and accommodate diverse learning needs.

## Learning pathways

Choose a pathway based on your role and experience level.

### Pathway 1: Engineer foundations

**For:** Software engineers new to AI code assistants

**Duration:** [TBD]

## Training by maturity level

Training approach varies based on team maturity classification.

### Starting teams

| Element | Approach |
|---------|----------|
| Primary format | Guided workshops (face-to-face preferred) |
| Facilitation | FDE-led with hands-on support |
| Pace | Slower, with more practice time |
| Focus | Foundations, building confidence |
| Support | Daily check-ins, immediate help available |
| Duration | 2-3 days of facilitated training |

**Recommended sequence:**
1. Day 1: Engineer Foundations workshop (full team)
2. Day 2: Hands-on practice with FDE support
3. Day 3: Tool-specific deep dive + independent practice

---

### Developing teams

| Element | Approach |
|---------|----------|
| Primary format | Blended (self-paced + facilitated) |
| Facilitation | FDE-supported workshops |
| Pace | Standard, self-directed where possible |
| Focus | Building proficiency, advanced techniques |
| Support | Weekly check-ins, scheduled clinics |
| Duration | Self-paced modules + half-day workshop |

**Recommended sequence:**
1. Week 1: Complete Engineer Foundations self-paced
2. Week 2: Advanced techniques workshop (half-day)
3. Week 3: Context Engineering self-paced
4. Week 4: Practice and consolidation

---

### Established teams

| Element | Approach |
|---------|----------|
| Primary format | Self-paced with optional workshops |
| Facilitation | On-demand consultation |
| Pace | Self-directed |
| Focus | Optimisation, advanced use cases |
| Support | Community channels, on-demand FDE |
| Duration | Self-paced, typically 4-6 hours total |

**Recommended approach:**
1. Complete relevant self-paced modules independently
2. Attend optional advanced workshops based on interest
3. Contribute learnings to community and repository

## Accessibility

All training materials are designed to be accessible and inclusive.

### Standards compliance

- WCAG 2.2 AA compliant
- GOV.UK content design standards
- Plain English (reading age 9 target)

### Accessibility features

| Feature | Implementation |
|---------|----------------|
| Screen reader compatibility | All materials tested with NVDA/JAWS |
| Captions | All videos captioned |
| Transcripts | Full transcripts for all audio/video |
| Keyboard navigation | All interactive elements keyboard accessible |
| Colour contrast | Minimum 4.5:1 ratio |
| Alt text | All images have descriptive alt text |
| Resizable text | Content reflows at 200% zoom |
| Reading level | Plain English, jargon explained |

### Neurodiversity support

| Accommodation | Availability |
|---------------|--------------|
| Extended time | Available for assessments |
| Quiet spaces | Provided for in-person workshops |
| Materials in advance | Sent 1 week before workshops |
| Breaks | Regular breaks built into workshops |
| Multiple formats | Same content in text, video, audio |
| Reduced visual clutter | Clean, minimal design |
| Progress saving | Self-paced modules save progress |

### Requesting accommodations

If you need accommodations not listed above:

1. Contact [PLACEHOLDER: accessibility contact]
2. Describe what would help you learn effectively
3. We'll work with you to find solutions

## Measuring training effectiveness

### Metrics collected

| Metric | Target | Purpose |
|--------|--------|---------|
| Training attendance | ≥70% of allocated users | Adoption indicator |
| Training satisfaction | ≥8.0/10 | Quality indicator |
| Module completion | ≥80% within 4 weeks | Engagement indicator |
| Knowledge check scores | ≥70% pass rate | Comprehension indicator |
| Time to proficiency | Baseline + improvement | Effectiveness indicator |

### Feedback collection

| Method | Timing | Purpose |
|--------|--------|---------|
| Post-module survey | After each module | Content quality |
| Workshop feedback | End of workshop | Facilitation quality |
| 30-day follow-up | 30 days after training | Application of learning |
| Quarterly review | Quarterly | Overall programme effectiveness |

### Continuous improvement

Training materials are updated based on:

- Feedback scores and comments
- FDE observations from engagements
- Common questions and misconceptions
- Tool updates and new features
- New patterns discovered in programme
